# FactorInterface 모듈

## 목적

**다양한 팩터를 통일된 인터페이스로 관리**하고, 멀티팩터 전략을 쉽게 구현할 수 있게 합니다.

다른 담당자(매크로, 펀더멘털, 기술적)가 개발한 팩터를 **동일한 방식으로 사용**할 수 있습니다.

---

## 핵심 개념

### 팩터(Factor)란?

주식의 수익률을 설명하거나 예측하는 데 사용되는 **특성 또는 지표**입니다.

| 팩터 유형 | 설명 | 예시 |
|-----------|------|------|
| **매크로** | 시장 전체 상황 | 금리, 환율, VIX |
| **펀더멘털** | 기업 재무 데이터 | ROE, PER, 부채비율 |
| **기술적** | 가격/거래량 패턴 | 이동평균, RSI, 모멘텀 |

---

## 핵심 기능

### 1. 통일된 인터페이스

모든 팩터는 동일한 메서드를 구현:
- `compute()` - 팩터 값 계산
- `get_signal()` - 매매 시그널 생성 (-1, 0, 1)

### 2. 예시 팩터 (기본 제공)

| 팩터 | 설명 | 매수 조건 | 매도 조건 |
|------|------|----------|----------|
| **SMA** | 이동평균 크로스오버 | 단기 > 장기 | 단기 < 장기 |
| **RSI** | 상대강도지수 | RSI < 30 (과매도) | RSI > 70 (과매수) |
| **Momentum** | 가격 모멘텀 | N일 수익률 > 0 | N일 수익률 < 0 |

### 3. 멀티팩터 결합

여러 팩터를 가중합하여 통합 시그널 생성:
- **가중합** (weighted_sum)
- **순위 평균** (rank_avg)
- **Z-score 합** (zscore_sum)

---

## 다른 담당자와의 협업

```
매크로 담당자    →  MacroFactor 구현   ─┐
펀더멘털 담당자  →  FundFactor 구현    ─┼→  FactorCombiner  →  통합 시그널
기술적 담당자    →  TechFactor 구현    ─┘
```

**규칙**: `BaseFactor`를 상속받아 `compute()`와 `get_signal()` 구현

---

## 시그널 해석

| 값 | 의미 | 행동 |
|----|------|------|
| **1** | 매수 신호 | 포지션 진입 |
| **0** | 관망 | 현 포지션 유지 |
| **-1** | 매도 신호 | 포지션 청산 |

## 입출력 요약

| 입력 | 출력 |
|------|------|
| OHLCV DataFrame | 팩터 값 Series |
| OHLCV DataFrame + 임계값 | 시그널 Series (-1, 0, 1) |
