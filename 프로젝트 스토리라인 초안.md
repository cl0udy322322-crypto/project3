# 프로젝트 스토리라인 초안

## 1. 문제 인식: “알파는 정말 존재하는가?”

### 1.1 문제 제기

- 많은 투자 전략이 기술적 지표 또는 단일 팩터에 의존
- 그러나 실제 시장에서 지속 가능한 알파는 제한적
- 특히 한국 시장(KOSPI200 / KOSDAQ150)에 특화된 알파 검증 연구는 부족

> 핵심 질문: 한국 주식시장에서 멀티 팩터 기반 접근은 실제로 알파를 만들어낼 수 있는가?

### 1.2 기존 접근의 한계

- 단일 지표 중심 전략 → 과적합 위험
- 기술적 분석 중심 → 시장 환경(매크로/펀더멘털) 반영 부족
- 전략 성과만 강조 → 알파의 구조적 원인 분석 부족

> 본 프로젝트는 전략 개발이 아니라 알파 탐색에 초점을 둠

## 2. 접근 방법: 멀티 팩터 기반 알파 탐색 프레임워크

### 2.1 분석 프레임워크 설계

본 프로젝트는 세 가지 축을 결합함
- 매크로 팩터 (Macro)
- 펀더멘털 팩터 (Fundamental)
- 기술적 팩터 (Technical)

### 2.2 역할 분담 구조

각 영역별 전문성을 분리하여 팩터 후보를 설계
- 매크로 분석 → 시장 레짐 및 필터
- 기본적 분석 → 종목 선별 팩터
- 기술적 분석 → 타이밍 시그널
- 전략 설계 → 멀티 팩터 결합
- 백테스트 → 알파 검증
- 보고서 → 스토리 구조화

> "팩터 생성 → 검증 → 결합 → 해석"의 연구 프로세스를 명확히 분리

## 3. 실행 과정: 알파 탐색의 단계적 수행

### 3.1 데이터 구축

- Universe: KOSPI200 + KOSDAQ150
- 데이터 소스:
  - 주가(FDR)
  - 재무(DART, KRX)
  - 매크로(FRED)
- 룩어헤드 방지 및 데이터 정합성 확보

### 3.2 팩터 후보 정의

**(1) 매크로 팩터**
- 금리, 환율, 인플레이션, 경기지표
- 레짐 분류(Risk-on / Risk-off)
- 시장과의 lag correlation 분석

**(2) 펀더멘털 팩터**
- Value / Growth / Quality
- Rank / Z-score 기반 스코어링

**(3) 기술적 팩터**
- Momentum / Trend / Volatility
- 매매 시그널 정의

> 팩터를 많이 만드는 것이 아니라, 의미 있는 팩터를 선별하는 과정

### 3.3 팩터 검증 및 필터링

- 단일 팩터 백테스트
- 성능 비교
- 최종 팩터 선정

> "모든 팩터가 알파를 만드는 것은 아니다"

### 3.4 멀티 팩터 전략 설계

**팩터 결합 구조:**
- 매크로 필터 → 시장 환경 결정
- 펀더멘털 랭킹 → 종목 선정
- 기술적 시그널 → 진입 타이밍

> 구조적 의미: 언제(매크로) + 무엇을(펀더멘털) + 어떻게(기술적)

### 3.5 백테스트 및 검증

- IS / OOS 테스트
- 벤치마크 비교
- 성과 지표:
  - CAGR, MDD, Sharpe, Alpha
- 레짐별 성과 분석

> 핵심 질문 재검증: 이 전략은 우연이 아니라 구조적 알파인가?

## 4. 결과: 알파의 존재 여부에 대한 답

### 4.1 주요 결과 요약

- 멀티 팩터 전략은 단일 팩터 대비 성능 개선
- 특정 시장 레짐에서 알파가 집중적으로 발생
- 펀더멘털 팩터의 기여도가 상대적으로 높음

> 핵심 메시지: 알파는 항상 존재하지 않고, 조건부로 나타난다.

### 4.2 알파의 구조적 해석

**알파 발생 구조:**
- 매크로 환경이 우호적일 때
- 펀더멘털 우량 종목이
- 기술적 추세와 결합될 때

> 알파 = 팩터의 조합 + 시장 환경의 함수

## 5. 시사점: 전략 개발이 아니라 '알파 탐색 방법론'의 확립

### 5.1 실무적 시사점

- 단일 전략보다 팩터 조합이 중요
- 매크로 필터는 필수 요소
- IS/OOS 분리의 중요성

### 5.2 방법론적 시사점

본 프로젝트의 진짜 성과는
- "전략 하나"가 아니라
- "알파를 찾는 프레임워크" 구축

## 6. 한계 및 향후 과제

### 6.1 한계

- 데이터 기간 제한
- 팩터 선택의 주관성
- 거래 비용 및 슬리피지 단순화

### 6.2 확장 방향

- 팩터 자동 선택 (ML)
- 동적 가중치 모델
- 해외 시장 적용
- 리스크 패리티 / 리스크 예산 모델 결합

## 7. 결론: 알파는 존재하는가?

**최종 답변:**
- 알파는 항상 존재하지는 않는다.
- 그러나 특정 조건과 구조 안에서는  
  탐색 가능하며, 설계할 수 있다.
